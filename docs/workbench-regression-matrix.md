# Workbench Regression Matrix

| Feature                        | Covered Tests                                                                            | Manual Acceptance Steps                                                                                         | Risk   | Status                    |
| ------------------------------ | ---------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- | ------ | ------------------------- |
| Claude provider detection      | `test:e2e:workbench` provider controls; release checklist CLI commands                   | Select Claude Code, inspect path/version/capabilities, run `/status`.                                           | High   | Covered                   |
| Codex provider regression      | `verify:mvp`, `test:integration`; manual smoke                                           | Select Codex, start a plan session, confirm streaming still works.                                              | High   | Covered                   |
| Session baseline               | init-claude API smoke; `test:e2e:workbench` init/discard paths                           | Start session in dirty repo and confirm baseline warning preserves pre-existing changes.                        | High   | Covered                   |
| Session-scoped diff            | `test:e2e:workbench` Diff tab checks                                                     | Open Diff after edit or `/init-claude`, refresh diff.                                                           | High   | Covered                   |
| Safe discard                   | `test:e2e:workbench` safe discard coverage                                               | Discard one file and all files; verify baseline dirty/untracked files remain.                                   | High   | Covered                   |
| Approval gate                  | `test:integration` task approvals; `test:e2e:workbench` ApprovalCard                     | Trigger `/init-claude apply` with ask rule, approve and reject.                                                 | High   | Covered                   |
| Permissions rules              | `test:e2e:workbench` create/disable/delete rule                                          | Run `/permissions`, add ask/deny rules, check hits.                                                             | High   | Covered                   |
| Shell Remote TUI authorization | `test:native-terminal-shell`, `native-terminal-broker.test.ts` shell authorization cases | Open Remote TUI, select Shell, confirm authorization prompt, verify wrong user/device/path tokens are rejected. | High   | Covered                   |
| init-claude                    | `test:e2e:workbench` ask approval refresh/reject/readonly                                | Scan, apply, approve, reject, and verify existing files are not overwritten.                                    | High   | Covered                   |
| Export                         | `test:e2e:workbench` export options                                                      | Run `/export`, copy/download Markdown with default and optional full diff/raw logs.                             | Medium | Covered                   |
| Commands                       | `test:e2e:workbench` Commands detail drawer                                              | Open Commands, inspect details, copy command, compare with logs.                                                | Medium | Covered                   |
| Compact                        | `test:e2e:workbench` inspector/context smoke                                             | Run `/compact`, confirm Workbench-local prompt context appears.                                                 | Medium | Covered                   |
| Usage                          | `test:e2e:workbench` actual/estimated usage badge                                        | Confirm Actual usage when provider emits usage; Estimated usage otherwise.                                      | Medium | Covered                   |
| Long logs                      | `verify:mvp` type/build; route limit review                                              | Load Logs with default limit, verify UI stays responsive.                                                       | Medium | Covered by bounded routes |
| Readonly mode                  | `test:e2e:workbench` readonly blocks init-claude write                                   | Mark session read-only, run `/init-claude apply`, confirm no files written.                                     | High   | Covered                   |
| Pending approval refresh       | `test:e2e:workbench` ApprovalCard hydrate                                                | Trigger approval, refresh browser, confirm card remains actionable.                                             | High   | Covered                   |
| Cancellation                   | `test:integration` cancel flow                                                           | Start a run, click Stop, verify logs/diff/export remain readable.                                               | Medium | Covered                   |
| Failed session logs            | `test:integration` timeout/failure flow                                                  | Force failure or approval timeout; inspect Logs and export.                                                     | Medium | Covered                   |

Notes:

- Best-effort command parsing depends on provider stream and tool payloads.
- Estimated usage is not billing data unless provider usage fields and pricing are configured.
- Workbench approval gate does not take over Claude Code official runtime tool approval.
- Shell Remote TUI authorization gates the interactive shell session before PTY creation; it does not approve every typed command after the shell is open.
